## 1. システム全体像

┌─────────────────────────────────────────┐

## 2. 毎週の作業

### STEP 1: 売上CSVをダウンロード

HPBシステム（ホットペッパービューティー管理画面）から売上データをCSV形式でダウンロードします。

**ダウンロード後の確認事項:**
- ファイル名を `sales.csv` に変更してください
- 文字コードは **UTF-8** を選択してください
- 列の順番が変わっていないか確認してください

---

### STEP 2: GitHubにアップロード

#### 方法A: ブラウザからアップロード（最も簡単）

1. ブラウザで `https://github.com/あなたのアカウント/salon-management-os` を開く

2. `data` フォルダをクリック

3. `sales.csv` をクリック

4. 右上の **鉛筆アイコン**（✏️）をクリック

5. 画面上部の「...」→「Upload file」を選択するか、  
   もしくは画面内の全テキストを選択（Ctrl+A）して削除し、  
   新しいCSVの内容を貼り付け

6. 下部の「Commit changes」ボタンをクリック

7. コメント欄に `2026年3月3週 売上更新` などと入力

8. 「Commit changes」をクリックして完了

---

#### 方法B: Git コマンドでアップロード（上級）

```bash
# 1. プロジェクトフォルダに移動
cd /Users/あなた/salon-management-os

# 2. 新しいCSVをdata/フォルダにコピー
cp ~/Downloads/sales.csv data/sales.csv

# 3. GitHubにアップロード
git add data/sales.csv
git commit -m "売上CSV更新 $(date '+%Y年%m月%d日')"
git push

# 完了！ 約1分後にダッシュボードが更新されます
```

---

### STEP 3: ダッシュボードを確認

アップロード後、約1〜2分待ってからダッシュボードを開いてください。

**URL**: `https://あなたのアカウント名.github.io/salon-management-os/`

ブラウザで **Ctrl+Shift+R**（Mac: Cmd+Shift+R）を押してキャッシュをクリアすると確実に最新データが表示されます。

## 3. 月初の作業

### targets.csv で目標を設定

毎月1日に、今月の目標をCSVに入力します。

**ファイルの場所**: `data/targets.csv`

**入力例（4月の目標設定）:**
```
month,store,new_target,sales_target,unit_price_target,contract_rate_target,continuation_rate_target
2026-04,大宮(直営),35,550000,15000,60,70
2026-04,川崎(直営),28,450000,14500,55,65
2026-04,池袋(直営),22,380000,13500,50,60
```

**各列の意味:**

| 列名 | 日本語 | 入力例 | 注意 |
|---|---|---|---|
| month | 対象年月 | 2026-04 | YYYY-MM形式 |
| store | 店舗名 | 大宮(直営) | sales.csvのZ列と完全一致 |
| new_target | 新規目標数 | 35 | 件数（整数） |
| sales_target | 売上目標 | 550000 | 円（整数、カンマなし） |
| unit_price_target | 平均単価目標 | 15000 | 円（整数） |
| contract_rate_target | 新規契約率目標 | 60 | %（整数、%マークなし） |
| continuation_rate_target | 継続契約率目標 | 70 | %（整数、%マークなし） |

## 4. ダッシュボードの見方

### タブ構成

```
📊全体KPI | 🏪店舗別 | 👤スタッフ | 🏆ランキング | 💎LTV管理 | 🚨アラート | 📋SVアクション
```

---

### 📊 全体KPI タブ

画面上部に表示される6つのKPIカード：

| カード | 見方 | 色の意味 |
|---|---|---|
| **総売上** | 全店舗合計売上 | 金色 = 目標達成率バッジで確認 |
| **新規数** | 全店舗合計新規客数 | 緑 = 80%以上達成 |
| **平均単価** | 全体の平均客単価 | — |
| **新規契約率** | 新規客が回数券を契約した率 | 緑≥80% / 黄≥60% / 赤<60% |
| **継続契約率** | 再来客が継続した率 | 同上 |
| **店舗数** | 稼働中の店舗数 | — |

**バッジの色:**
- 🟢 緑 = 目標80%以上達成（良好）
- 🟡 黄 = 目標60〜79%（要注意）
- 🔴 赤 = 目標60%未満（要対策）

---

### 🏪 店舗別 タブ

各店舗のKPIカードと一覧表が表示されます。

**達成率の進捗バー:**
- 緑のバー = 80%以上達成
- 黄のバー = 60〜79%達成
- 赤のバー = 60%未満（要対策）

**フィルターボタン** でクリックすると特定の店舗だけ表示できます。

---

### 👤 スタッフ タブ

スタッフ別のKPIが一覧で確認できます。

| 列名 | 内容 |
|---|---|
| スタッフ名 | 施術担当者名 |
| 所属店舗 | 担当店舗 |
| 新規数 | 担当した新規客数 |
| 売上 | 担当した売上合計 |
| 平均単価 | 客単価の平均 |
| 新規契約数 | 新規客への回数券契約数 |
| 新規契約率 | 新規客への契約率 |
| 継続契約率 | 再来客への継続率 |

---

### 🏆 ランキング タブ

4種類のランキングが表示されます：

1. **💴 売上ランキング** — 売上が多いスタッフTop5
2. **📝 契約率ランキング** — 新規契約率が高いスタッフTop5  
3. **💰 平均単価ランキング** — 客単価が高いスタッフTop5
4. **✨ 新規獲得ランキング** — 新規客数が多いスタッフTop5

🥇🥈🥉 でTop3が表示されます。

---

### 💎 LTV管理 タブ

回数券を購入した顧客のLTV（顧客生涯価値）管理画面です。

**ステータスの見方:**
- ✓ 正常 (緑) = 30日以内に来店
- 要フォロー (黄) = 30〜60日未来店
- ⚠ 離脱リスク (赤) = 60日以上未来店

---

### 🚨 アラート タブ

**HPB集客アラート**: 新規目標80%未満の店舗が表示されます  
**顧客離脱アラート**: 未来店の回数券顧客が表示されます

各アラートに「推奨アクション」が表示されるので、それに従って対応してください。

---

### 📋 SVアクション タブ

KPIの状況に応じて、SV（スーパーバイザー）が取るべきアクションが自動生成されます。

**カードの枠の色:**
- 🔴 赤枠 = 緊急対応が必要
- 🟡 黄枠 = 要注意・改善が必要
- 🔵 青枠 = 情報・軽微な改善

## 5. Slackレポートの確認方法

### レポートの送信タイミング

**毎週月曜日 午前9時**に指定のSlackチャンネルに自動送信されます。

### レポートの読み方

```
━━━━━━━━━━━━━━━━━━━━
📊 週次サロンレポート
2026年3月16日（月）
━━━━━━━━━━━━━━━━━━━━

■ 全体KPI
💴 総売上: ¥1,250,000
✨ 新規: 32件 (目標75件)
📝 平均契約率: 55%
         ↑ここを毎週チェック

━━━━━━━━━━━━━━━━━━━━
■ 店舗別進捗
🏪 大宮(直営)
  売上: ¥500,000 (83%)  ← 達成率
  新規: 15件/30件 (50%) ← 目標比
  契約率: 60% | 継続率: 72%
...
```

### レポートが届かない場合

1. GitHubリポジトリの「Actions」タブを開く
2. 最新のワークフロー実行をクリック
3. エラーメッセージを確認する
4. 最もよくある原因: `SLACK_WEBHOOK_URL` の設定漏れ

**手動で送信する場合:**
1. GitHub → Actions タブ
2. 「週次サロンレポート自動送信」をクリック
3. 「Run workflow」ボタンをクリック
4. 「Run workflow」を確認

## 6. HPBアラート対応マニュアル

### アラートの基準

| アラート | 条件 | 緊急度 |
|---|---|---|
| 🔴 新規不足 | 目標達成率 70%未満 | 高 |
| 🟡 要強化 | 目標達成率 70〜79% | 中 |
| ✅ 正常 | 目標達成率 80%以上 | — |

---

### 対応フロー

```
HPBアラート発生
      │
      ▼
【確認】何が原因か？
      │
      ├─ 新規が減った → クーポン・掲載改善
      ├─ 競合が増えた → 差別化強化  
      └─ 口コミが減った → 口コミ施策

      ▼
【短期対策】今週中に実施
  ① HPBクーポンの追加・価格改定
  ② 掲載写真・テキストの更新
  ③ スタッフからお客様への口コミ依頼

      ▼
【中期対策】2〜4週間で実施
  ① 新規クーポン企画
  ② SNS連動キャンペーン
  ③ Google口コミの強化
```

---

### 具体的な対策メニュー

| 施策 | 詳細 | 効果が出るまで |
|---|---|---|
| クーポン追加 | 初回限定¥2,980など新クーポン | 即日〜1週間 |
| 掲載写真更新 | 施術中・ビフォーアフター写真 | 1〜2週間 |
| 口コミ依頼 | 来店客へQRコードで誘導 | 2〜4週間 |
| プロフィール改善 | 強みを前面に出した文章改訂 | 1〜2週間 |
| 期間限定セール | 月内限定価格でCTR改善 | 即日〜3日 |

## 8. SVマネジメント運用方法

毎週月曜日にSlackレポートを確認後、「SVアクション」タブを開きます。

**🔴 緊急（今週中に対応）**

| 状況 | アクション |
|---|---|
| 契約率 50%未満 | ①SV本人がカウンセリングに同席 ②トークを聞いて改善点をフィードバック ③ロープレを実施 |

| 状況 | アクション |
|---|---|
| 契約率 50〜60% | ①成功事例をチームに共有 ②カウンセリングシートを見直し |

## 9. 初期セットアップ手順

### 必要なもの

- GitHubアカウント（無料）
- SlackのWebhook URL（管理者から入手）
- 売上CSVデータ

---

### セットアップ手順（詳細）

#### A. GitHubアカウント作成（初回のみ）

1. `https://github.com` を開く
2. 「Sign up」をクリック
3. メールアドレス・パスワードを設定
4. メール認証を完了

---

#### B. リポジトリを作成

1. GitHubにログインした状態で右上「+」をクリック
2. 「New repository」をクリック
3. 以下を入力：
   - Repository name: `salon-management-os`
   - Visibility: `Public`（無料のGitHub Pagesに必要）
4. 「Create repository」をクリック

---

#### C. ファイルをアップロード

1. 作成したリポジトリページで「uploading an existing file」をクリック
2. このシステムの全ファイル・フォルダをドラッグ&ドロップ  
   （フォルダ構成を保ったままアップロード）
3. 「Commit changes」をクリック

---

#### D. GitHub Pagesを有効化

1. リポジトリの「Settings」タブをクリック
2. 左メニューの「Pages」をクリック
3. 「Source」で **Deploy from a branch** を選択
4. Branch: **main**、フォルダ: **/ (root)** を選択
5. 「Save」をクリック
6. 数分後にURLが表示されます

**URL形式**: `https://あなたのユーザー名.github.io/salon-management-os/`

---

#### E. Slack Webhookを設定

**Slack App作成:**
1. `https://api.slack.com/apps` を開く
2. 「Create New App」→「From scratch」
3. App Name: `Salon Report Bot`
4. ワークスペースを選択 → 「Create App」
5. 「Incoming Webhooks」→ オンに切り替え
6. 「Add New Webhook to Workspace」
7. 通知を送りたいチャンネルを選択
8. Webhook URLをコピー（`https://hooks.slack.com/services/...`）

**GitHub Secretsに登録:**
1. GitHubリポジトリの「Settings」
2. 「Secrets and variables」→「Actions」
3. 「New repository secret」
4. Name: `SLACK_WEBHOOK_URL`
5. Secret: コピーしたURL
6. 「Add secret」

## 10. トラブルシューティング

### よくある問題と解決方法

---

**問題: ダッシュボードが真っ白/表示されない**

原因と対策:
- GitHub Pagesの設定を確認（Settings→Pages）
- URLが正しいか確認（`https://ユーザー名.github.io/salon-management-os/`）
- リポジトリがPublicになっているか確認

---

**問題: データが古いまま更新されない**

対策:
1. ブラウザで `Ctrl+Shift+R`（Mac: `Cmd+Shift+R`）でハードリロード
2. GitHubのリポジトリで `data/sales.csv` が最新になっているか確認
3. GitHub Actionsでエラーが出ていないか確認

---

**問題: Slackにレポートが届かない**

原因と対策:
1. **SLACK_WEBHOOK_URLが未設定**: GitHub → Settings → Secrets → Actionsで確認
2. **ワークフローが実行されていない**: GitHub → Actionsタブで確認
3. **手動実行でテスト**: Actions → 「週次レポート」→「Run workflow」

---

**問題: 売上数値がおかしい**

よくある原因:
- sales.csvの列がずれている（最も多い原因）
- CSVの文字コードがShift-JIS（UTF-8に変換が必要）
- 金額列（L列）に文字が入っている

確認方法: ExcelやGoogleスプレッドシートでCSVを開いて列を確認

---

**問題: 「オンダリフト新規」の数が合わない**

計算ルール:
```
オンダ新規数 = I列に「オンダ」を含む && V列=「新規」 && !I列に「オンダリフト」 && M列≠0
```

CSVのI列（項目名）とM列の値を確認してください。

---

**問題: 新しい店舗を追加したい**

手順:
1. `data/store_master.csv` に新店舗を追加
2. `data/targets.csv` に新店舗の目標を追加
3. sales.csvのZ列（Y列店舗名）が新店舗名と一致していることを確認
4. GitHubにアップして完了

---

*このマニュアルに記載されていない問題が発生した場合は、システム管理者にお問い合わせください。*
